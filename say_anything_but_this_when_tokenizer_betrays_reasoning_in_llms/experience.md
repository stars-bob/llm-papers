# 实验设计详解

## 任务设计

### Tokenization-Consistency Probe（分词一致性探测）

该探测任务的设计核心在于隔离分词器与模型推理之间的交互影响：

**任务定义**: 给定带有标记跨度的文本和替换每个标记跨度为替代词的指令，模型必须：
1. 识别目标跨度
2. 应用编辑
3. 保留所有非目标文本

**关键特性**: 该任务评估模型是否认识到不同的词元ID组合可以解码为相同的字符串，因此尽管内部表示不同，但代表相同的语义。这种设置隔离了源于分词器离散字母表与模型学习规则之间交互的失败。

### 实验流程

```
原始文本 → [Selector] → 标记目标词（括号包围） → Tokenizing → 词元ID序列 → LLM → 输出词元ID → Detokenizing → 输出文本
```

1. **选择器（Selector）**: 从每篇文档中随机采样p%的非停用词作为替换目标
2. **标记**: 将目标词用括号包围，生成带注释的文本版本
3. **预处理**: 在文档前添加明确的替换指令
4. **分词**: 通过分词器将输入映射为词元ID序列
5. **模型生成**: LLM基于输入词元ID生成输出
6. **反分词化**: 将输出词元ID解码为文本

## 评估模型

共评估10个最先进的开源LLM，跨越4个主要模型家族：

### Gemma系列（Google）
| 模型 | 参数规模 | 分词器类型 |
|------|---------|-----------|
| Gemma3-270M | 0.27B | SentencePiece (Unigram) |
| Gemma3-1B | 1B | SentencePiece (Unigram) |
| Gemma3-4B | 4B | SentencePiece (Unigram) |
| Gemma3-12B | 12B | SentencePiece (Unigram) |

**分词器特点**: 采用Unigram语言模型通过SentencePiece实现，使用超过256k词元的广泛词汇表支持大规模多语言能力。

### Llama系列（Meta）
| 模型 | 参数规模 | 分词器类型 |
|------|---------|-----------|
| Llama3.2-1B | 1B | Byte-Level BPE |
| Llama3.2-3B | 3B | Byte-Level BPE |
| Llama3.1-8B | 8B | Byte-Level BPE |

**分词器特点**: 采用Byte-Level BPE，从单个字节构建词元以实现多语言文本和代码的无损编码，同时缓解字符级BPE固有的词汇爆炸问题。

### Mistral系列
| 模型 | 参数规模 | 分词器类型 |
|------|---------|-----------|
| Ministral-8B | 8B | Tekken (BPE变体) |
| MistralSmall-24B | 24B | Tekken (BPE变体) |

**分词器特点**: 实现"Tekken"分词器，专为源代码和欧洲语言的压缩效率优化的专用BPE变体。

### Qwen系列（阿里巴巴）
| 模型 | 参数规模 | 分词器类型 |
|------|---------|-----------|
| Qwen3-4B | 4B | Byte-Level BPE |
| Qwen3-30B | 30B | Byte-Level BPE |

**分词器特点**: 采用Byte-Level BPE，与Llama系列类似。

## 评估指标

### 三类结果分类

| 类别 | 词元ID变化 | 表面文本变化 | 含义 |
|------|-----------|-------------|------|
| **Unchanged** | ❌ 相同 | ❌ 相同 | 模型未执行替换 |
| **Replaced** | ✅ 不同 | ✅ 不同 | 成功替换 |
| **Different** | ✅ 不同 | ❌ 相同 | 幻影编辑（分词器诱导） |

### 关键量化指标

1. **幻影编辑率**: "Different"类别占总试验的比例
2. **替换成功率**: "Replaced"类别占总试验的比例
3. **子词分段转换**: 输入子词数 → 输出子词数的热力图分析

## 干预实验：Token-ID掩码

### 方法

**问题**: 实际修正分词器缺陷需要修改分词规则和词汇表，然后重新训练模型以内化修订后的词元ID映射，这在学术环境中由于计算限制通常不可行。

**干预措施**: 采用轻量级事后干预——掩码在"Different"实例中经验性触发幻影编辑的特定输出词元ID。

**特点**:
- 不修改分词器、嵌入空间或模型学习表示
- 仅约束解码空间，使模型无法为观察到的"Different"样本中的违规词元分配概率质量
- 强制模型生成替代输出

### 结果

| 模型家族 | 掩码前"Different"率 | 掩码后"Different"率 | 改善幅度 |
|---------|-------------------|-------------------|---------|
| Gemma3 | 7.8%-16.4% | 0.0%-5.4% | 显著下降 |
| Llama3.x | 7.2%-12.3% | 0.4%-5.1% | 显著下降 |
| Mistral | 9.7%-11.3% | 1.8%-3.9% | 显著下降 |
| Qwen3 | 5.9%-13.4% | 0.6%-2.7% | 显著下降 |

**关键发现**: 
- "Different"失败模式被显著抑制，百分比下降至约0-5%
- "Unchanged"率也降低，表明抑制幻影编辑词元迫使模型重新归一化概率分布，提高了真正正确替换的机会
- 剩余的小比例"Different"表明，移除不良词元序列无法保证完全解决问题，因为分词的基本多对一映射属性仍然存在

## 错误类型分类

### 八种系统性分词器伪影

| 错误类型 | 描述 | 示例 |
|---------|------|------|
| **Error 1: Whitespace-boundary shift** | 词汇表重复存储带/不带前导空格的词 | "February" ↔ " February" |
| **Error 2: Whitespace detachment/reattachment** | 带前导空格的词元被分割为空格词元+词元 | [" Saturday"] → [" ", "Saturday"] |
| **Error 3: Newline/whitespace substitution** | 空格前缀词元转换为换行词元+无空格词元 | [" However"] → ["\n", "However"] |
| **Error 4: Intra-word resegmentation** | 原子词元被替换为组成子词的序列 | [" unbelievable"] → ["un", "bel", "ievable"] |
| **Error 5: Proper-noun segmentation ambiguity** | 专有名词的替代分段重新分配字符跨越词元边界 | [" Jub'", "ilee"] → ["J", "ub", "ilee"] |
| **Error 6: Morphological boundary surfacing** | 形态学边界显现，词被分段为词缀 | [" repaid"] → ["re", "paid"] |
| **Error 7: Acronyms split** | 首字母缩略词以不同大写单元组装 | [" HIV"] → ["H", "IV"] |
| **Error 8: Plural/possessive tail tokens** | 复数/所有格形式被重分段为基础词元+"s" | [" rights"] → ["right", "s"] |

## 关键实验洞察

### 子词分段转换热力图（Gemma3-4B）

| 输入子词数 \ 输出子词数 | 1 | 2 | 3 | 4 | 5 | 6+ |
|---------------------|---|---|---|---|---|----|
| 1 | 72.2% (最频繁) | - | - | - | - | - |
| 2 | 少量 | - | - | - | - | - |
| 3 | - | - | 中等 | - | - | - |
| 4 | - | - | - | 中等 | - | - |
| 5 | - | - | - | - | 最亮区域（长词） | - |

**主要发现**:
- (1→1)单元占所有"Different"实例的约72.2%——大多数幻影编辑保持子词计数不变，仅改变词元ID身份
- 这表明空白变体词元对（仅前导空格不同的相同词）的存在是主要失败机制
- 78%的案例保持相同子词计数，19.7%增加计数（分割），仅2.3%减少计数（合并）
- 模型很少成功"修复"分词器引入的次优分段

## 实验结论

1. **分词器是隐藏的推理瓶颈**: 部分推理失败源于分词器层面，而非模型参数不足
2. **规模化并非万能**: 增加模型参数无法系统性地解决源于分词器诱导的表示不匹配问题
3. **模型具备潜在能力**: 当幻影编辑路径被阻断时，模型表现出改进的推理能力，表明 tokenizer-induced phantom edits 构成了概率景观中的"最小阻力路径"
4. **需要分词器级修复**: 在投入训练更大模型和更大语料库之前，应考虑分词器级别的补救措施
