## 研究背景

现代大语言模型（LLM）普遍采用带有PreNorm的残差连接（Residual Connections）作为基础构建块。标准的残差更新公式为：

```
hl = hl-1 + fl-1(hl-1)
```

这种设计虽然通过恒等映射为梯度提供了直接的传播路径，使得深层网络的训练成为可能，但存在一个根本性问题：**所有层输出都以固定的单位权重累加**。这种均匀的聚合方式导致隐藏状态幅度随深度呈O(L)增长， progressively diluting each layer's contribution（逐渐稀释每一层的贡献）。

在PreNorm范式下，早期层的信息被淹没且无法被选择性检索；实践中，很大一部分层可以被剪枝而几乎不损失性能。这与RNN在序列维度上面临的挑战类似——在注意力机制出现之前，RNN也无法有效地选择性地访问历史信息。

---

## 核心贡献

### 1. Attention Residuals (AttnRes)

作者提出了**注意力残差（Attention Residuals）**，将固定的残差累加替换为对前面层输出的**softmax注意力加权聚合**：

```
hl = Σ(i=0 to l-1) αi→l · vi
```

其中：
- `αi→l` 是通过softmax计算的学习得到的注意力权重
- `vi` 是第i层的输出（v0为token嵌入）
- 每一层只需要一个可学习的d维伪查询向量 `wl`

关键设计：**伪查询wl是与层的前向计算解耦的独立学习参数**，这使得注意力权重可以在不等待顺序输出的情况下并行计算。

### 2. Block Attention Residuals

Full AttnRes需要在每一层访问所有前面层的输出，导致O(Ld)的内存和通信开销。为了在超大规模模型训练中使用，作者引入了**块注意力残差（Block AttnRes）**：

- 将L层划分为N个块（block）
- 块内使用标准残差累加，将块内所有层输出求和为单个表示
- 块间使用Full AttnRes，只对N个块级表示进行注意力计算

**复杂度降低**：从O(Ld)降至O(Nd)

**经验发现**：N≈8时即可恢复大部分Full AttnRes的收益。

### 3. 基础设施优化

为了在实际大规模训练中高效部署Block AttnRes，作者开发了以下系统优化：

#### 跨阶段缓存（Cross-stage Caching）
在流水线并行中，每个物理阶段处理多个虚拟阶段。通过缓存之前接收的块表示，可以避免重复传输，将每次传输的开销从O(C)降至O(P)。

#### 两阶段计算策略（Two-phase Computation）
- **Phase 1**: 批量计算块间注意力，将所有S=L/N个查询批量处理
- **Phase 2**: 顺序计算块内注意力，并通过online softmax合并结果

这使得每层内存访问成本仅为(N/S + 3)d读取和2d写入，端到端推理延迟开销<2%。

#### 内存高效预填充
对于长上下文（如128K tokens），通过序列维度的张量并行分片，将每设备内存占用从15GB降至约1.9GB（8路TP）。

## 实验结果

### 缩放定律实验

在5个模型规模（194M到528M激活参数）上进行训练，拟合幂律曲线 L = A × C^(-α)：

| 方法 | 拟合公式 | 在5.6 PFLOP/s-days的loss |
|------|----------|-------------------------|
| Baseline | L = 1.891 × C^(-0.057) | 1.714 |
| Block AttnRes | L = 1.870 × C^(-0.058) | 1.692 |
| Full AttnRes | L = 1.865 × C^(-0.057) | ~1.691 |

**关键发现**：Block AttnRes达到1.692的loss，相当于Baseline需要1.25×的计算量才能达到相同性能。

### 48B参数模型预训练

在Kimi Linear架构（48B总参数/3B激活参数）上进行1.4T tokens预训练：

#### 训练动态
- **验证loss**: AttnRes在整个训练过程中始终低于Baseline，差距在decay阶段扩大
- **输出幅度**: Baseline存在PreNorm稀释问题——深层必须学习越来越大的输出才能保持影响力；Block AttnRes将增长限制在块内，呈现有界的周期性模式
- **梯度幅度**: Baseline的早期层梯度异常大；Block AttnRes的可学习softmax权重引入了源之间的竞争，梯度分布更加均匀

#### 下游任务性能

| 任务类别 | 任务 | Baseline | AttnRes | 提升 |
|----------|------|----------|---------|------|
| 通用推理 | MMLU | 73.5 | 74.6 | +1.1 |
| 通用推理 | GPQA-Diamond | 36.9 | 44.4 | +7.5 |
| 通用推理 | BBH | 76.3 | 78.0 | +1.7 |
| 数学代码 | GSM8K | 81.7 | 82.4 | +0.7 |
| 数学代码 | HumanEval | 59.1 | 62.2 | +3.1 |
| 数学代码 | Minerva Math | 53.5 | 57.1 | +3.6 |
| 中文理解 | CMMLU | 82.0 | 82.9 | +0.9 |
| 中文理解 | C-Eval | 79.6 | 82.5 | +2.9 |

**观察**：在多步推理任务（GPQA-Diamond、Math、HumanEval）上的提升尤为显著，这与"改善的深度信息流有利于组合任务"的假设一致。

### 消融实验

在16层模型上验证关键设计选择：

| 变体 | Validation Loss |
|------|-----------------|
| Baseline (PreNorm) | 1.766 |
| DenseFormer | 1.767 |
| mHC | 1.747 |
| **Full AttnRes** | **1.737** |
| Block AttnRes (S=4) | 1.746 |
| w/ input-dependent query | 1.731 |
| w/ sigmoid (替代softmax) | 1.741 |
| w/o RMSNorm | 1.743 |

**关键结论**：
1. **输入依赖性至关重要**：DenseFormer使用固定标量权重，无收益；mHC引入输入依赖，有改进；AttnRes使用内容依赖的选择，效果最好
2. **Softmax优于Sigmoid**：softmax的竞争性归一化强制更锐利的源选择
3. **RMSNorm至关重要**：防止输出幅度大的层或块主导注意力权重
4. **多头注意力反而有害**：表明最优的深度混合在通道间基本统一

### 块大小分析

从S=1（Full AttnRes）到S=32的扫描显示：
- S=2, 4, 8的loss都接近1.746
- S=16, 32向Baseline靠拢
- 实际采用N≈8个块，平衡性能和基础设施效率

### 架构偏好分析

在固定计算量（~6.5×10^19 FLOPs）和参数预算下，扫描25种(dmodel/Lb, H/Lb)配置：

- Baseline最优在dmodel/Lb≈60（1.847）
- AttnRes最优在dmodel/Lb≈45（1.802）

**结论**：AttnRes更能有效利用额外深度，在更深、更窄的网络上表现更好。

### 学习到的注意力模式

可视化16层模型的注意力权重分布：
1. **保留局部性**：每层主要关注直接前驱，但也出现选择性off-diagonal集中（如第4层关注早期源）
2. **层专业化**：嵌入h1始终保留非平凡权重，尤其在pre-attention层；pre-MLP输入更依赖近期表示
3. **Block AttnRes保留结构**：对角优势、嵌入持续性、层专业化都得以保留

## 理论视角

### 序列-深度对偶性

残差连接通过固定递推在深度上传播信息，类似于RNN在时间上传播信息。作者通过结构矩阵视角统一分析：

- **标准残差**：M是下三角全1矩阵
- **Highway网络**：M是1-半可分的，具有输入依赖权重
- **(m)HC**：M是m-半可分的，对应矩阵值状态的深度线性注意力
- **AttnRes**：M是稠密的、秩为L的，执行深度softmax注意力

这一视角揭示：**现有残差变体实际上是深度轴上的线性注意力实例，而AttnRes将其推广到softmax注意力**，完成了与序列建模相同的"线性到softmax"过渡。

---

## 结论

Attention Residuals通过用学习到的、输入依赖的深度注意力替代固定、均匀的残差累加，解决了PreNorm稀释问题。Block AttnRes通过块级压缩将内存和通信从O(Ld)降至O(Nd)，结合跨阶段缓存和两阶段计算策略，在大规模训练中的开销微乎其微（训练<4%，推理<2%）。

实验表明，AttnRes在所有评估的模型规模上都带来一致的改进，在48B参数模型的1.4T tokens预训练中，下游任务全面优于Baseline，尤其在多步推理任务上提升显著。

随着未来硬件互联能力的提升，采用更细粒度的块大小甚至Full AttnRes代表了进一步改进性能的自然路径。
