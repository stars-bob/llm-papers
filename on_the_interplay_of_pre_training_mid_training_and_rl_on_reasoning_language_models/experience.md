## 数据集设计

### 数据生成框架
基于 **GSM-Infinite** 框架构建可控合成推理数据集，包含三个关键组件：

#### 1. 依赖图（Dependency Graphs）
- 每个推理问题表示为有向图 G = (V, E)
- 节点 v ∈ V 对应变量
- 有向边 e ∈ E 表示变量间依赖关系
- 图以指定的答案节点 v* 结束

#### 2. 推理复杂度控制
```
op(G) = |E|  (边的数量 = 算术操作数)
```
- 控制任务难度从基础算术到复杂多步推理

#### 3. 上下文渲染（Contextual Rendering）
- 预定义上下文模板 τ（如：动物园-动物、学校-教师）
- 将依赖图 G 渲染为完整数学问题

### 数据分布

#### 预训练（Pre-training）
| 操作数范围 | 占比 | 说明 |
|------------|------|------|
| op=2-4 | 20% | 基础操作 |
| op=5-7 | 30% | 中等难度 |
| op=8-10 | 50% | 较复杂操作 |
| **总计** | **10B tokens** | **ID范围** |

#### 中训练（Mid-training）
- 目标：在op=11-14范围内建立先验
- 数据配比：20% op=2-10（防灾难性遗忘）+ 80% op=11-14
- 与预训练相同的目标函数（next-token prediction）

#### 后训练/RL（Post-training）
- **算法**：GRPO (Group Relative Policy Optimization)
- **数据量**：200K samples
- **实验变体**：
  - op=7-10 (ID)
  - op=9-12 (混合)
  - op=11-14 (边缘)
  - op=17-20 (困难)

## 训练配置

### 预训练设置
```
数据量：10B tokens
操作范围：op=2-10
目标：掌握基础推理原语
```

### 中训练设置
```
批次大小：512 × 1024 tokens
序列长度：2048
目标函数：next-token prediction (与预训练相同)
数据分布：窄化到模型新兴但不完整能力的范围
```

### RL设置（GRPO）
| 参数 | 值 |
|------|-----|
| **算法** | GRPO |
| **批次大小** | 1024 |
| **rollout倍数 (r)** | 6 |
| **序列长度 (L_total)** | 2048 |
| **参考模型** | γ = 1（启用） |

### 计算预算等效公式

#### RL计算成本
```
C_RL = (8 + 2γ) × P × N × r × L_total

其中：
- P = 模型参数
- N = RL样本数
- r = rollout大小 (6)
- L_total = 总序列长度 (2048)
- γ = 1（启用参考模型）
```

#### 等效中训练token成本
```
T_RL_eq = (4/3 + γ/3) × N × r × L_total
        = 5/3 × N × r × L_total  （当γ=1时）
```

## 评估协议

### 过程验证评估框架

#### 评估步骤
1. 模型生成自由格式解决方案
2. 解析为预测依赖图 Ĝ 和最终答案 â
3. 步骤级评估：对每个gold节点 v ∈ V 比较预测与真实节点、依赖关系和数值
4. 过程准确率 = 所有gold节点的平均步骤级准确率

#### 正确性标准
```
Correctness = ProcessAcc(Ĝ, G) == 1 AND â == a*
```
- 仅当推理步骤和最终答案都正确时才视为完全正确
- 所有pass@k指标（pass@1, pass@128）基于此严格标准

### 泛化维度

#### 1. 外推泛化（Extrapolative/Depth）
评估模型解决比训练期间更复杂问题的能力
- **ID**：op=2-10（预训练范围）
- **OOD-edge**：op=11-14（略微超出）
- **OOD-hard**：op=15-20（大幅超出）

#### 2. 上下文泛化（Contextual/Breadth）
评估模型将推理技能迁移到表面形式不同但底层逻辑相同的新领域的能力
- 上下文A（如动物园-动物）：预训练主要暴露
- 上下文B（如学校-教师）：预训练长尾暴露

## 关键实验结果

### 实验1：RL有效性条件

| 任务类型 | RL数据范围 | pass@128改进 |
|----------|------------|--------------|
| ID (op=2-10) | op=7-10 | 无改进 |
| OOD-edge (op=11-14) | op=11-14 | 显著提升 |
| OOD-hard (op=15-20) | op=11-14 | **+42%** |

**结论**：RL在能力边缘数据上训练时产生真正的能力提升。

### 实验2：预训练暴露对上下文泛化的影响

| 上下文B暴露比例 | pass@128 (上下文B) | 结果 |
|----------------|-------------------|------|
| 0% | 接近0 | 失败 |
| 0.1% | 接近0 | 失败 |
| **1%** | 显著提升 | **成功** |
| 10% | 高 | 良好 |

**结论**：仅需1%的预训练暴露即可支持RL的上下文泛化。

### 实验3：中训练+RL的协同效应

| 配置 | OOD-edge性能 | OOD-hard性能 |
|------|-------------|-------------|
| Full Mid | 高pass@1 | 低 |
| Full RL | 中 | 中 |
| Light-RL | **最高pass@1** | 中 |
| Heavy-RL | 中 | **最高pass@128** |

**结论**：中训练+RL组合优于单独使用任一方法。

### 实验4：过程奖励效果

| 奖励混合 | op=2-14 | op=15-20改进 |
|----------|---------|-------------|
| 1.0 Outcome | 基准 | - |
| 0.2 Out + 0.8 Proc | +0.0% | **+4.3%** |
| 0.5 Out + 0.5 Proc | -0.1% | +3.4% |
| 1.0 Out if Proc Correct | +0.3% | **+4.1%** |

**结论**：过程验证奖励减少奖励破解，提升推理保真度。

## 关键经验与教训

1. **RL不是万能药**：RL只能在预训练留下足够探索空间时产生真正的新能力

2. **能力边缘是关键**：设计RL数据时应针对模型pass@1失败但pass@k成功的任务

3. **中训练被低估**：中训练阶段在现代LM管道中扮演关键但未被充分探索的角色

4. **稀疏暴露足够**：预训练中仅需约1%的特定上下文暴露即可支持后续RL泛化

5. **过程奖励重要**：整合过程级验证到奖励函数可有效缓解奖励破解

6. **计算预算分配策略**：
   - ID可靠性优先：80%中训练 + 20%RL
   - OOD探索优先：20%中训练（建立先验）+ 80%RL
